informatica 筆記

  • 可以一次呼叫不同的數據庫
  • 可以對相同的資料做整合

Middleware

  • Informatica 可以連接各種源代碼,應用程序源，包括:

  • SAP認證的數據集成工具

  • 可將數據推拉到 SAP R3 和 SAP BW 系統。

  • 大多數應用程序源都有連接適配器。

  • SAPR3、SAP等兩個應用程序之間的中間件BW等。

Extract

  • 從源系統中選擇數據倉庫所需的數據的過程
  1. 文件、數據庫或其他應用程序也可以使用。
  2. 可能會引起某種程度的清潔或變形。
  3. 建立後可重複自動化
  • PowerCenter從數據庫中的表（或相關表組）或文件中逐行讀取數據。
  • 此數據庫或文件稱爲源。
  • 源結構包含在源定義對象中。

Transform

  • Transformation
  1. 更改或轉換導入數據倉庫的數據。
  2. 通常以"清潔"和"一致"數據爲目的。
  • cleansing (清潔)
  1. 從導入數據倉庫的數據中刪除錯誤和不一致性的過程
  2. 可能包含多個階段
  • PowerCenter 將行轉換爲第二個（目標）系統可用的格式。
  • 此轉換的邏輯在轉換對象中定義。

Load

  • PowerCenter 將數據逐行寫入數據庫中的表（或相關表單）或文件。
  • 將此數據或是資料夾 提取到 目標資料庫。
  • A目標結構必須包含在專案目標定義裡。

Mapping

  • 邏輯定義 ETL 進程
  • 將數據從一個或多個源移動和轉換到另一個源的一系列轉換,一個或多個目標
  • 映射完全在PowerCenter內部

import from database

  • 選擇帶入的數據庫

  • 如果找不到自己的庫 按三個點尋找自己相對應的庫
  • 輸入新庫的host、port、Service name
  • 按下TEST Connect，輸入庫名跟密碼，測試好正常再按確定
  • 用好連結的庫AUG22就會出現在data souce裡面
  • 輸入剛剛User、庫明、密碼 按下Connect
  • 按下表單會自動跳出內部資料
  • 新增新user 這三個全選all
  • system 也是全選 只有SYSDBA、SYSOPER不要

如果你要取消關聯性

  • 點選標頭

  • 把想取消的關聯性key Type 改成(NOT A KEY)
